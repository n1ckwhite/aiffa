# Git pull vs fetch — на практике и без мифов

## Что делает `fetch`

`fetch` обновляет удалённые ссылки (`origin/main`, etc.), **не трогая** твою рабочую ветку.

```bash
git fetch origin
git log --oneline --decorate --graph --all | head
```

## Что делает `pull`

`pull` = `fetch` + попытка влить изменения в текущую ветку (merge или rebase — зависит от настроек).

```bash
git pull --rebase
```

## Когда что использовать

### fetch

- Хочешь безопасно “подсмотреть” изменения
- Нужно сравнить ветки перед интеграцией

### pull

- Ты уверен, что готов влить изменения прямо сейчас

## Заключение

`fetch` — безопасный шаг для обзора. `pull` — шаг для интеграции.
