# Почему компонент рендерится 100 раз: чек-лист

## Чек-лист

### 1) Проверь `useEffect`

- Нет ли `setState` без условий?
- Корректны ли зависимости?

### 2) Проверь новые объекты/функции

```ts
const buildConfig = (value: number) => ({ value }); // новый объект на каждый вызов
```

### 3) Локализуй источник

Временно упростить дерево и найти компонент, который триггерит обновления.

## Заключение

Рендер-лупы лечатся дисциплиной: корректные эффекты, стабильные props и понятная архитектура данных.
