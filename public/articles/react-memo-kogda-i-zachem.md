# React.memo: когда он реально нужен (и когда мешает)

## Как работает `memo`

### Базовый пример

```tsx
import React from "react";

type Props = { value: number };

export const Counter = React.memo(({ value }: Props) => {
  return <div>{value}</div>;
});
```

## Когда помогает

- Тяжёлый компонент
- Стабильные props
- Ререндеры идут из-за родителя

## Когда мешает

- Props всегда новые (новые объекты/функции на каждый рендер)
- Компонент лёгкий и мемоизация дороже

## Заключение

Сначала — данные/архитектура/стабильность props. Потом — `memo` как точечная оптимизация.
