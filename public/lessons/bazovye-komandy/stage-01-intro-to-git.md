#  Введение в Git

git — это современная распределённая система контроля версий, которая позволяет отслеживать изменения в коде, работать над проектом в команде и сохранять историю всех изменений.

С помощью git разработчики могут создавать «снимки» состояния проекта, возвращаться к предыдущим версиям, работать параллельно над разными задачами и объединять результаты работы. Git делает процесс командной разработки прозрачным и безопасным, позволяя избежать потери данных и конфликтов при слиянии изменений.

##  TLDR

| Понятие | Коротко |
| --- | --- |
| VCS (система контроля версий) | Хранит историю изменений и помогает работать в команде |
| Коммит | Снимок изменений с сообщением «что/почему» |
| Ветка | Параллельная линия разработки для безопасных экспериментов |
| Push/Pull | Push отправляет ваши коммиты на сервер, Pull подтягивает чужие |

---

##  Что нужно уметь после этапа

- Пояснить, что такое VCS и зачем она нужна
- Объяснить разницу между локальным и удалённым репозиторием
- Описать жизненный цикл изменений: working tree → staging → commit → push
- Понимать, что такое ветка и зачем она нужна

---

##  Полезные команды для старта

```bash
# инициализация и базовые настройки
git init
git config --global user.name "Ваше Имя"
git config --global user.email you@example.com

# создание первого коммита
echo "hello" > README.md
git add README.md
git commit -m "init"
```

---

##  Частые ошибки новичков

- Коммит «всё подряд»: коммитьте логически связанные изменения
- Отсутствие `.gitignore`: мусорные файлы попадают в историю
- Работа в `main` без веток — сложнее поддерживать историю

<details>
<summary>[ГЛУБОКОЕ ПОГРУЖЕНИЕ] Снимки vs дельты | как Git хранит историю</summary>

- Git хранит коммиты как снимки состояния файлового дерева, а не как патчи; повторно использует неизменённые блобы
