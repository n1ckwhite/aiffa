#  Инструменты для разрешения конфликтов

##  Какие инструменты помогают в разрешении конфликтов?

При работе с Git существует множество инструментов, которые значительно упрощают процесс разрешения конфликтов. От встроенных утилит до мощных визуальных редакторов — каждый разработчик может выбрать наиболее удобный для себя инструмент.

---

##  TLDR

| Класс | Инструменты | Когда выбирать |
| --- | --- | --- |
| Встроенные | git mergetool, git diff, git status | Базовая терминальная работа |
| IDE/UI | VS Code, GitKraken, SourceTree | Визуальное разрешение конфликтов |
| Дифф‑утилиты | Meld, KDiff3, Beyond Compare | Трёхпанельный дифф, сложные кейсы |

---

###  1. Встроенные инструменты Git

1. **git mergetool** — встроенный механизм Git для запуска внешних инструментов сравнения
2. **git diff** — показывает различия между версиями в командной строке
3. **git status** — отображает список конфликтующих файлов

####  Примеры команд

```bash
git mergetool
```

```bash
git diff
```

```bash
git status
```

---

###  2. Популярные визуальные инструменты

1. **Visual Studio Code** — встроенная поддержка Git, удобный интерфейс сравнения, подсветка конфликтов, кнопки быстрого разрешения
2. **KDiff3** — трёхпанельный вид, авто‑разрешение простых конфликтов, подробная визуализация
3. **Meld** — интуитивный интерфейс, подсветка синтаксиса, редактирование в процессе сравнения
4. **Beyond Compare** — профессиональный инструмент с мощным сравнением и поддержкой разных форматов

####  Примеры команд

```bash
git config --global merge.tool kdiff3
```

```bash
git config --global merge.tool meld
```

```bash
git config --global merge.tool bc3
```

---

###  3. Процесс разрешения конфликтов с инструментами

1. **Настройте инструмент** (например, Meld) и параметры запуска
2. **Запустите процесс разрешения** через выбранный инструмент
3. **Разрешите конфликты в UI** (левая панель — ваши изменения, правая — входящие, центр — итог)
4. **Сохраните результат** и зафиксируйте изменения в Git

####  Примеры команд

```bash
# Установка Meld как инструмента по умолчанию
git config --global merge.tool meld
# Автоматический запуск без подтверждения
git config --global mergetool.prompt false
```

```bash
git mergetool
```

```bash
git add .
git commit -m "Resolve merge conflicts"
```

---

###  4. Рекомендации по выбору инструмента

- **Для начинающих**: Visual Studio Code или Meld
- **Для продвинутых**: Beyond Compare или KDiff3
- **Для работы в терминале**: vimdiff
- **Для больших команд**: инструмент, принятый в команде

---

##  Советы по использованию

- Настройте инструмент до возникновения конфликтов
- Изучите горячие клавиши выбранного инструмента
- Регулярно обновляйте инструменты
- Создавайте резервные копии перед разрешением сложных конфликтов
