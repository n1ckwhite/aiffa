#  Изменение порядка во Flexbox: свойство `order` и доступность

Свойство `order` позволяет менять визуальный порядок элементов **без изменения HTML**. Полезно в адаптивных интерфейсах, но важно учитывать доступность и логику фокуса.

---

##  Как работает `order`

- Каждому flex-элементу можно задать `order` (целое число, по умолчанию `0`).
- Элементы с меньшим `order` появляются раньше.
- Допустимы отрицательные значения (`-1`, `-2`, …).

```css
.container { display: flex; gap: 10px; }
.a { order: 2; }
.b { order: 1; }
.c { order: 3; }
```

**Визуально:** `.b` → `.a` → `.c`.

---

##  Примеры использования

###  Перестановка карточек на мобильном
```css
@media (max-width: 600px) {
  .sidebar { order: 2; }
  .content { order: 1; }
}
```

###  Перемещение CTA-блока вверх
```css
.cta { order: -1; }
```

> Если у элементов одинаковый `order`, сохраняется исходный DOM-порядок.

---

##  Доступность, навигация с клавиатуры и чтение экранными дикторами

- **DOM-порядок не меняется**. Читатели экрана, таб-навигация и фокус следуют DOM-порядку, а не визуальному.
- Сильное расхождение между визуальным и DOM-порядком может сбивать пользователей.
- Для критичных перестановок лучше **менять HTML-порядок** или тщательно тестировать клавиатурную навигацию.

> Не используйте `order` для «логической» перестановки форм, шагов или навигации — это может нарушить ожидаемую последовательность фокуса.

---

##  Частые приёмы и тонкости

- Сбрасывать `order` удобно через классы-утилиты (`.order-0`, `.order-1`, `.order--1`).
- Комбинируется с `flex-wrap`: порядок действует в пределах каждой строки, затем по строкам.
- При динамической фильтрации/сортировке часто лучше управлять DOM, а не `order`.

##  Итог

- `order` меняет только визуальное расположение. Навигация и чтение по-прежнему следуют DOM.
- Используйте для косметических/адаптивных перестановок; для логики — меняйте HTML-порядок.
