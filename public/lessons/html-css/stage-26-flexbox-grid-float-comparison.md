#  Flexbox vs Grid vs Float: когда что выбирать

Три подхода к макетам решают разные задачи. Ниже — краткая шпаргалка, примеры и практические рекомендации по выбору инструмента.

---

##  Flexbox
- **Что это:** Одноосная модель (главная/поперечная оси).
- **Сильные стороны:** Ряды/колонки, выравнивание, распределение пространства, адаптивные карточки.
- **Слабые стороны:** Неудобен для сложных двумерных сеток.
- **Типичные кейсы:** Меню, тулбары, карточки, формы в строку, медиа-объекты.

```css
.menu {
  display: flex;
  gap: 12px;
  justify-content: space-between;
  align-items: center;
}
```

---

##  Grid
- **Что это:** Двухосная модель (строки и колонки).
- **Сильные стороны:** Сетки любой сложности, областьные макеты, контроль обеих осей, автозаполнение.
- **Слабые стороны:** Порог входа выше, старые браузеры могут не поддерживать.
- **Типичные кейсы:** Макеты страниц (header/sidebar/main/footer), галереи, дашборды.

```css
.layout {
  display: grid;
  grid-template-columns: 200px 1fr 200px;
  grid-template-rows: 60px 1fr 60px;
  gap: 16px;
}
```

---

##  Float
- **Что это:** Исторически — обтекание текста вокруг медиаконтента.
- **Сильные стороны:** Почти универсальная совместимость.
- **Слабые стороны:** Нужен clearfix, тяжёлая поддержка макетов, не для современных интерфейсов.
- **Типичные кейсы:** Обтекание изображений в статьях.

```css
.img-left {
  float: left;
  margin: 0 20px 20px 0;
}
```

---

##  Сравнение по признакам

| Критерий         | Flexbox                       | Grid                                | Float                       |
|------------------|-------------------------------|-------------------------------------|-----------------------------|
| Оси              | 1 ось                         | 2 оси                               | 1 ось (обтекание)          |
| Сложность сетки  | Низкая/Средняя                | Любая                                | Низкая                      |
| Адаптивность     | Отличная                      | Отличная                            | Ограниченная                |
| Контроль пробелов| `gap`, выравнивание           | `gap`, треки, области               | Маргины, хаки               |
| Совместимость    | Современные браузеры          | Современные браузеры                | Все                         |
| Лучшие кейсы     | Ряды, меню, карточки          | Страницы, гриды, дашборды           | Обтекание изображений       |

---

##  Руководство по выбору

- **Линейная компоновка** (одна ось, выравнивание, гибкие карточки) → Flexbox.
- **Двумерная сетка** (и строки, и колонки важны) → Grid.
- **Текст + обтекание картинки** → Float.
- **Комбинирование**: Flex внутри ячеек Grid — распространённая практика (Grid задаёт области, Flex выравнивает внутри).

> Избыточное смешение подходов усложняет поддержку. Держите каждый инструмент в своей «зоне ответственности».

##  Итог

Выбор между Flexbox, Grid и Float зависит от задачи: **для рядов — Flexbox, для сеток — Grid, для обтекания — Float**. Часто оптимально сочетать Grid для общей структуры и Flex для выравнивания внутри областей.
